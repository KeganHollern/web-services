# Advanced Script Variable Hiding

I needed a new way to hide global variables from being accessed — or even found, for that matter — by threads that aren't created by me.
This is useful for anticheats because it lets them access and modify values from multiple threads without exposing those values to any malicious thread.

The code is a little dense, so I'll do my best to walk through it.

---

```sqf
_GLOBAL_ARRAY = [[], []];
_get = {
    params ["_GLOBAL_ARRAY", "_data"];
    private ["_name", "_default", "_index"];
    _name = _data select 0;
    _default = _data select 1;
    _index = (_GLOBAL_ARRAY select 0) find toLower(_name);
    if (_index == -1) exitWith {_default};
    (_GLOBAL_ARRAY select 1) select _index;
};
_set = {
    params ["_GLOBAL_ARRAY", "_data"];
    private ["_name", "_value", "_index"];
    _name = _data select 0;
    _value = _data select 1;
    _index = (_GLOBAL_ARRAY select 0) find toLower(_name);
    if (_index == -1) then {
        (_GLOBAL_ARRAY select 0) pushBack toLower(_name);
        (_GLOBAL_ARRAY select 1) pushBack _value;
    } else {
        (_GLOBAL_ARRAY select 1) set [_index, _value];
    };
};

[_GLOBAL_ARRAY, ["test", true]] call _set;

[_GLOBAL_ARRAY, _get, _set] spawn {
    params ["_GLOBAL_ARRAY", "_get", "_set"];
    while{true} do {
        waitUntil{[_GLOBAL_ARRAY, ["test", false]] call _get};
        systemchat "true";
        uiSleep 1;
        [_GLOBAL_ARRAY, ["test", false]] call _set;
    };
};
[_GLOBAL_ARRAY, _get, _set] spawn {
    params ["_GLOBAL_ARRAY", "_get", "_set"];
    while{true} do {
        waitUntil{!([_GLOBAL_ARRAY, ["test", true]] call _get)};
        systemchat "false";
        uiSleep 1;
        [_GLOBAL_ARRAY, ["test", true]] call _set;
    };
};
```

The key to the entire system is `_GLOBAL_ARRAY`. It stores the values of our _global variables_ — I'll keep referring to them that way since they essentially behave the same.

`_get` and `_set` do all the heavy lifting, manipulating `_GLOBAL_ARRAY` so the rest of the system works.

`_get` is the simplest by far, so I'll explain it first. It looks up the variable name in the name list, and if the variable doesn't exist it returns the default value passed in `_data`. If it's found, it returns the value from the parallel value list.

`_set` does the harder work. It needs to modify the global value without creating a second instance of the array — no copying, just mutating. If the name isn't in the name list, push the name onto the name list and the value onto the value list. If it is, use `set` to overwrite the value in place.

Below those functions is the first test I ran on this. Works perfectly — every second it flips the value of `"test"` from true to false.

I consider this some of my best work. A neat little function that's very useful for anticheats.
