# What do cheaters use on ArmA 3 servers?

What kind of attacks do cheaters actually use on Arma 3 servers, and what are servers most often vulnerable to?
I'm going to break it down into a few categories and cover each separately, with examples drawn directly from my own work.

Heads up: I'm only covering scripts in this post. Memory cheats are more involved and I'll save those for a separate post.

---

The first category is _self-boosting_ scripts — the most basic stuff: "No Recoil", "Unlimited Ammo", "God Mode". These are the cheats anticheats and `scripts.txt` filters were designed to catch.

No recoil is straightforward. [InfiSTAR] blocks it, so any server running InfiSTAR doesn't really need to worry about it. Some mods like _Desolation_ legitimately modify a unit's recoil to be more realistic, and cheaters lean on this to gain an edge in firefights.

```sqf
player setUnitRecoilCoefficient 0;
```

Unlimited ammo is harder to detect. InfiSTAR catches the obvious cases, but adding a small delay to the loop is enough to slip past it:

```sqf
while{true} do {
    (vehicle player) setVehicleAmmo 1;
    uiSleep 2;
};
```

The most effective ban is a `scripts.txt` filter — adding `5 setVehicleAmmo` will catch most of the cheats currently in circulation.

God mode is also tricky because there are several different ways to do it. Here are a few, ordered roughly from most to least effective:

```sqf
player allowDamage false;

player addEventHandler ["HandleDamage", {false}];

while{true} do {
    player setDamage 0;
};
```

None of these have an easy detection path. Once again the most reliable defense is a `scripts.txt` filter.

---

The next category is _trolling_ scripts — still pretty basic, but the goal is to mess with the rest of the server. "Teleport All" and "Explode All" both live here.

Teleport all is common, and there are a few ways to write it. I use this one:

```sqf
{
    _x setPos (getPos player);
    uiSleep 0.1;
} forEach allPlayers;
```

BattlEye hoped a `setPos` count restriction would stop these. They were wrong — `uiSleep 0.1;` defeats it. It's on the server to filter this. Teleport all is genuinely poisonous to a community: if it keeps happening, players just stop showing up.

Explode all also has multiple flavors. Arma has _wonderful_ networking behavior where local explosions damage remote units, so you can spawn an explosive object locally on every player's position:

```sqf
{
    "HelicopterExploSmall" createVehicleLocal (getPos _x);
    uiSleep 0.1;
} forEach allPlayers;
```

This one is actually dumb on Bohemia's part — `createVehicleLocal` isn't logged by `createVehicle.txt`, so the only real defense is, again, `scripts.txt`.

---

The third category is _remote execution_ — by far the most complex of the bunch. These exploit a server's networking to run code on other players or on the server itself: "Kill Target", "Freeze Target", "Admin Disabler", and so on.

Remote executions are born from holes in a server's networking and BattlEye filters. There are too many variants to cover individually, but the most common one I've seen is `createUnit` remote execution. Patching it comes down to `remoteexec.txt`. Without strong remote-exec filters, a server is essentially guaranteed to have these scripts run on it.

---

The last category is _server destruction_ scripts — the nastiest of the bunch. Anything from "Crash Server" to "Database Injection" lives here. The goal is to ruin the server and make it unplayable.

Crashing a server is most often done by mass desync. Desync can be triggered so many ways that detecting it directly is very difficult. In practice, a combination of `createVehicle` and `scripts.txt` filters tends to ban the cheaters using these tools long before they pull the desync lever — the rest of their cheat trips other filters first.

Database injection started as a proof of concept I built for _Altis Life_. When it lands, it can take a server down for weeks. If you keep noticing the same script running on your server every time you log in, check your database.

---

That's the menu. Knowing what cheaters actually use is the first step to actually stopping them.
