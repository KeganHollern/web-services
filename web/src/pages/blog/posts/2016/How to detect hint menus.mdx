# How to Detect Hint Menus

The worst nightmare for any anticheat developer is coming across a cheat that's nearly impossible to detect.
Hint menus are one of those — there's no way to disable the `hint` command, and no built-in way to read what's currently being displayed.
So how would an anticheat go about stopping people from using these things?

![Aether Hack hint menu listing options like God Mode, Infinite Ammo, and No Fatigue](/blog/how-to-detect-hint-menus/img-1.png)

The easiest and simplest answer is strict [BattlEye] filters. If the cheater can't run their menu in the first place, there's nothing to worry about.

---

What if you wanted to do more than kick — actually _ban_ for using a hint menu? That's harder, and it requires shipping a mod. The only way to read the contents of a hint at runtime is to modify the `RscHint` class at `configFile >> "RscInGameUI" >> "RscHint"` and add an `onLoad` handler:

```cpp
onLoad="uiNamespace setVariable ['displayhint',_this select 1]";
```

That stashes a reference to the hint display in the UI namespace the moment it loads. From there, an anticheat running on the client can grab the display, walk its controls, and pull text out of the structured-text control:

```sqf
while {true} do {
    _display = uiNamespace getVariable ['displayhint', displayNull];
    if (!isNull _display) then {
        _structuredText = _display displayCtrl 102;
        _text = toLower (ctrlText _structuredText);

        //--- check _text for bad strings (like "god mode")
    };
    uiSleep 1;
};
```

Once `_text` is lowercased, it's just substring matching against whatever signatures you care about — names of common menus, cheat features (`god mode`, `infinite ammo`, `esp`), or anything else that consistently shows up in popular hint-based hacks.

---

That's it. Hint menus go from "nearly impossible to detect" to trivial the moment you have a mod component on the client — the entire cheat is rendering itself in plain text into a control you now have a handle to.
