> So I have been working on a new project with my [Raspberry Pi 3](https://www.raspberrypi.org/products/raspberry-pi-3-model-b/). I was looking for situations where something running on my current PC would block an application like [Wireshark](https://www.wireshark.org/), and how to work around this protection. My solution was a direct bridge through my Raspberry Pi along my ethernet cable.

First things first. The Raspberry Pi 3 only has 1 ethernet port, so I had to purchase an adapter for a second one. [I purchased this one on amazon and had no troubles](https://www.amazon.com/AmazonBasics-1000-Gigabit-Ethernet-Adapter/dp/B00M77HLII/ref=sr_1_1?dd=9MLP35dHsdPPzlSttU5YeQ%2C%2C&ddc_refnmnt=pfod&ie=UTF8&qid=1487196158&sr=8-1&keywords=USB%2Bto%2Bethernet&refinements=p_97%3A11292772011&th=1), it is the same speed as the onboard port the Pi has. This could be accomplished with IP Tables and using the Wlan0 interface, however I opted to spend money and make my life easier.

The first step is to enable SSH and connect your Pi to wifi. [Here is a tutorial for that if you don't already know.](https://www.raspberrypi.org/documentation/remote-access/ssh/)

Next you will need to connect over [Putty](http://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html), The default login is: `pi` / `raspberry`

Good now you can save that file with `Ctrl+O` and exit with `Ctrl+X`

Now you can shutdown your Pi using `sudo shutdown`.

**Hardware setup time!**

![img](https://ss.lystic.zip/Legacy/hnhjEyl2/photo_2017-03-04_16-01-29.jpg)

The red X's are unneeded for this build, I was using them during initial setup and never removed them.
The yellow circle represents the eth1 interface, this connects to your router/switch/ect.
The green circle represents the eth0 interface, this connects to your PC.
The purple circle is the power connector, I directly connected this to a USB port on my PC so the Pi boots whenever I boot my PC.

Once all of this is connected, You should be able to boot your pi and use your internet as if nothing was changed.

**Now the software :D**

For this example, I will be using TCPDump, you can install this with `sudo apt-get install tcpdump`

![img](https://ss.lystic.zip/Legacy/Oy6qX7yu/FNfiKX4pUc5JeGzw.png)

First verify that your bridge is setup. (You wouldn't have internet on your PC if it wasn't)
For this I used ifconfig, however it can be done many ways.

![img](https://ss.lystic.zip/Legacy/3QQHpbKd/KQDIYbcDZsWJq09k.png)

Now you can start monitoring packets

You can cancel the packet capture with `Ctrl+C`

**Extension**
So if your internet is faster than 100Mbps the raspberry pi may not be your best bet. If you need Between 100Mbps and 1Gbps I would recommend doing something similar to this with the new [Asus Tinker board](https://www.asus.com/uk/Single-board-Computer/TINKER-BOARD/) which has onboard 1Gbps ethernet.

I plan to continue this project with a Raspberry Pi Zero, [Power over Ethernet](http://lifehacker.com/power-a-raspberry-pi-zero-over-ethernet-1752284809), and [libtins](http://libtins.github.io/).
