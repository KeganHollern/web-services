# Hardware Packet Monitor - Raspberry Pi

> My solution was a direct bridge through my Raspberry Pi along my ethernet cable.

I have been working on a new project with my [Raspberry Pi 3].
I was looking for situations where something running on my current PC would block an application like [Wireshark], and how to work around this protection.
My solution was a direct bridge through my Raspberry Pi along my ethernet cable.

First things first. The Raspberry Pi 3 only has 1 ethernet port, so I had to purchase an adapter for a second one.
I purchased this [Amazon adapter] and had no troubles, it is the same speed as the onboard port the Pi has.
This could be accomplished with IP Tables and using the `wlan0` interface, however I opted to spend money and make my life easier.

---

The first step is to enable SSH and connect your Pi to wifi.
[Here is a tutorial for that if you don't already know.][Raspberry Pi SSH]

Next you will need to connect over [Putty].
The default login is: `pi` / `raspberry`

![shell terminal](/blog/hardware-packet-monitor-raspberry-pi/img-1.png)

Once connected, you'll need to navigate to `/etc`

```bash
cd /etc
```

Now, you'll need to edit the `rc.local` file using nano.

```bash
sudo nano rc.local
```

You'll need to add the following to the file.

```bash
ifconfig eth0 0.0.0.0
ifconfig eth1 0.0.0.0
brctl addbr br0
brctl addif br0 eth0
brctl addif br0 eth1
ifconfig br0 up
```

This zeroes out both ethernet interfaces, creates a bridge `br0`, adds both interfaces to it, and brings the bridge up. All traffic between `eth0` and `eth1` now passes through transparently.

I added a bit more when playing around. The new file should look like this.

![full rc.local file content](/blog/hardware-packet-monitor-raspberry-pi/img-2.png)

Good, now you can save that file with `Ctrl+O` and exit with `Ctrl+X`.

Now you can shutdown your Pi using `sudo shutdown`.

---

**Hardware setup time!**

![picture of raspberry pi all connected](/blog/hardware-packet-monitor-raspberry-pi/img-3.jpg)

The red X's are unneeded for this build, I was using them during initial setup and never removed them.
The yellow circle represents the `eth1` interface, this connects to your router/switch/etc.
The green circle represents the `eth0` interface, this connects to your PC.
The purple circle is the power connector, I directly connected this to a USB port on my PC so the Pi boots whenever I boot my PC.

Once all of this is connected, you should be able to boot your Pi and use your internet as if nothing was changed.

---

**Now the software**

For this example, I will be using **TCPDump**, you can install this with `sudo apt-get install tcpdump`.

```bash
sudo apt-get install tcpdump
tcpdump --help
```

First verify that your bridge is setup. You wouldn't have internet on your PC if it wasn't.
For this I used `ifconfig`, however it can be done many ways.

```bash
ifconfig
```

![ifconfig output](/blog/hardware-packet-monitor-raspberry-pi/img-4.png)

Now you can start monitoring packets.

```bash
sudo tcpdump -i br0 -X -n not port 22
```

This listens on the bridge interface, prints hex and ASCII output, skips DNS resolution, and filters out SSH traffic so you don't flood the output with your own Putty session.

<VideoPlayer src="/blog/hardware-packet-monitor-raspberry-pi/img-5.mp4" />

You can cancel the packet capture with `Ctrl+C`.

---

If your internet is faster than 100Mbps the Raspberry Pi may not be your best bet.
If you need between 100Mbps and 1Gbps I would recommend doing something similar to this with the [Asus Tinker Board] which has onboard 1Gbps ethernet.

I plan to continue this project with a Raspberry Pi Zero, [Power over Ethernet], and [libtins].
